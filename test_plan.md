
# План тестирования

## Цели тестирования
1. Проверить соответствие реализованного функционала требованиям (функциональное тестирование)
2. Оценить удобство и понятность интерфейса реальными пользователями (юзабилити-тестирование)
3. Выявить баги и узкие места до сдачи проекта

## Типы тестирования
- Функциональное (ручное)
- Юзабилити-тестирование (5–7 одногруппников)
- Кросс-браузерное (Chrome, Firefox, Edge, Safari мобильная)

## Среда тестирования
- ОС: Windows 11 / macOS 15 / Ubuntu 24.04
- Браузеры: Chrome 129+, Firefox 130+, Edge 129+
- Мобильные: Chrome iOS, Safari iOS
- Инструменты: GitHub Issues, Google Forms, Loom (запись экрана при необходимости)
